Migrations run under BEGIN IMMEDIATE with a 5s busy timeout, recheck the version under the write lock, create a consistent SQLite backup, and record each version in the same transaction. Nested application transactions use savepoints.
Durable Lifecycle & Retrieval
Canonical durable memory
Candidates remain extraction records and point to promoted_memory_id.
Durable memories own current, superseded, and retracted lifecycle state.
Validity windows, status reason, timestamp, and generation prevent accidental reactivation.
Conflict control
Normalized subject_key groups memories about the same subject.
Audits create idempotent potentially-contradicts review relations.
Review never automatically retracts or supersedes a memory; supersession is atomic and cycle-checked.
Offline-first retrieval
FTS5 is the default and remains fully usable without a provider or network.
Default memory search returns current durable memories; lifecycle filters reveal historical states.
Results may expose lexical rank, semantic rank, and fused score metadata.
Optional hybrid index
Only source chunks and current promoted memories are embedded.
Float32 cosine ranking is fused with FTS through reciprocal-rank fusion.
Endpoint, model, and dimension fingerprints never mix in one ranking operation.
Evidence sync commits first. Embedding jobs then move through pending → complete / failed; failures stay retryable and never roll back sync or disable FTS. Remote endpoints require explicit opt-in and receive redacted text only.
Code-Aware Summary & Source Repair
1 · Select safe context
→ Git-tracked and non-ignored files only
→ Manifest entrypoints → memory links → recent commits
→ Stable path ordering; reject external, symlink, binary, generated, dependency, and credential paths
→
2 · Bound + fingerprint
→ Maximum 12 files, 20,000 chars per file, 120,000 total
→ Record path, full content hash, truncation, and selection reason
→ Include the user-owned project-summary-notes.md overlay
→
3 · Protect output
→ Input fingerprint drives staleness
→ Output hash detects manual summary edits
→ Normal refresh refuses overwrite; --force backs up first
Adapter parse failures are deduplicated by adapter, path, and error code with sanitized messages, attempt counts, first/last occurrence, and resolution time. A later successful parse marks the failure resolved; inspect them through sources failures or list_source_failures.
Privacy, Retention & Recovery Boundary
Redact before persistence
API keys, bearer tokens, private keys, credential URLs, and provider-specific secrets are removed at the storage boundary.
Project literal/regex policies add domain-specific redaction without a raw bypass.
The same redactor protects vectors, exports, regenerated summaries, errors, and logs.
Administrative privacy CLI
privacy audit / export / import / scrub / delete / prune
Exports are redacted by default; raw export requires explicit CLI confirmation and is not exposed through MCP.
Retention supports per-adapter maximum age plus per-source overrides.
Evidence-aware mutation
Scrub atomically rebuilds source chunks, FTS, relations, quality state, and embeddings.
Delete removes dependent candidates, reassesses remaining evidence, and retracts orphaned durable memories.
Hashed tombstones stop later sync or deterministic extraction from resurrecting deleted sources.
Verified recovery + auditability
Sensitive mutations use a temporary SQLite backup and verify quick_check, foreign keys, and logical references.
Success removes the temporary backup; failure restores and reports the retained path.
The operation log records migrations and administrative actions without raw content or search queries.
