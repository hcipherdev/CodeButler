# Code Butler v0.4 Optional Hybrid Retrieval Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add privacy-safe, optional embedding storage and hybrid retrieval while preserving byte-for-byte-compatible FTS behavior whenever hybrid retrieval is disabled or unavailable.

**Architecture:** Keep SQLite FTS as the synchronous source of truth and add an independent embedding subsystem behind focused provider, persistence, build, and ranking modules. Evidence writes commit before embedding work begins; embedding jobs are retryable and vectors are isolated by endpoint/model/dimension fingerprint. The existing MCP search handler becomes async only at the orchestration boundary and returns the current FTS results unchanged on every disabled or unavailable-provider path.

**Tech Stack:** TypeScript, Node.js 24 built-in `node:sqlite`, native `fetch`, Vitest, FTS5, Float32 vector blobs, pure-JavaScript cosine similarity and reciprocal-rank fusion.

**Branch constraint:** Work directly on the branch active when this conversation started (`main`). Do not create a worktree or switch branches because the user explicitly requires this unless they say otherwise.

## File Structure

- `src/types.ts`: public retrieval, embedding, queue, vector, status, and rank metadata types.
- `src/config.ts`: backward-compatible defaults and parsing for `retrieval`, `embeddings`, and `privacy`.
- `src/storage/migrations.ts`: ordered v4 embedding schema migration.
- `src/storage/store.ts`: embedding owner enumeration, queue transitions, vector persistence/invalidation, and eligible-vector reads.
- `src/embeddings/provider.ts`: OpenAI-compatible `/embeddings` adapter, endpoint classification, privacy gate, response validation, and sanitized errors.
- `src/embeddings/fingerprint.ts`: stable content hashes, endpoint hashes, provider fingerprints, and Float32 blob conversion.
- `src/embeddings/service.ts`: eligible owner discovery, queue reconciliation, batched builds/retries, status reporting, and lifecycle-safe invalidation.
- `src/search/hybrid.ts`: cosine scoring and reciprocal-rank fusion for chunk and current-memory results.
- `src/search/service.ts`: FTS-first retrieval orchestration and exact fallback behavior.
- `src/sync/service.ts`: post-commit embedding queue/build trigger that never changes sync success.
- `src/mcp/tools.ts`: async hybrid search wiring and additive rank metadata exposure without renaming tools.
- `src/cli.ts`: `embeddings build` and `embeddings status` commands.
- `src/doctor/service.ts`: embedding configuration, queue, coverage, and fingerprint/model checks.
- `evaluation/retrieval.json`: checked-in evidence fixture and expected evidence IDs.
- `scripts/eval-retrieval.ts`: deterministic Recall@K/MRR evaluator.
- `package.json`: `eval:retrieval` script.
- `README.md`, `MEMORY_SYSTEM.md`, `docs/index.md`, `docs/quickstart.md`, `docs/mcp-setup.md`, `docs/architecture.md`, `docs/public/architecture.html`: Release 3 configuration, privacy, operations, and fallback documentation.
