# Code Butler v0.2 Reliability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Deliver Release 1 reliability foundations without changing existing CLI or MCP behavior.

**Architecture:** Move schema evolution and transaction control into focused storage modules while retaining the synchronous `MemoryStore` API. Add additive audit and doctor metadata, strengthen citation resolution against exact stored chunks and repository-contained files, and introduce an unwired privacy redaction service.

**Tech Stack:** TypeScript ESM, Node.js 24 `node:sqlite`, Vitest.
