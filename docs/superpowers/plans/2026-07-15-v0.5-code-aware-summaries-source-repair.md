# Code Butler v0.5 Code-Aware Summaries and Source Repair Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make project summaries inspect a deterministic, privacy-safe selection of tracked implementation files, preserve user-owned notes and manual summary edits, and persist actionable source parsing failures until they are resolved.

**Architecture:** Add a focused project-summary code selector that obtains candidates only from Git-tracked files, resolves every candidate beneath the repository real path, applies strict content/path exclusions, and returns a deterministic bounded manifest. Extend summary metadata with an output-content hash and add a notes overlay so refresh can distinguish input staleness from manual output edits. Add an ordered SQLite migration and store/service APIs for deduplicated source failures, then connect conversation adapters, CLI, MCP, Doctor, and documentation without changing existing command/tool requirements.

**Tech Stack:** TypeScript, Node.js 24 (`node:fs`, `node:path`, `node:child_process`, `node:sqlite`), Git plumbing commands, Vitest, MCP SDK, VitePress.

**Branch constraint:** Work directly on the branch active when this conversation started (`main`). Do not create a worktree or switch branches because the user explicitly requires this unless they say otherwise.

- `src/project-summary/service.ts`: summary input composition, notes overlay, fingerprints, output hash/manual-edit detection, forced backup, and status/result metadata.
- `src/project-summary/providers.ts`: provider payload receives the selected implementation manifest and user notes through existing generator input types.

- `src/sources/codex.ts`: structured parser outcomes and source-failure persistence/resolution for Codex and Claude logs.
- `src/cli.ts`: `sources failures [--json]` and manual-summary refresh/status reporting.
- `src/mcp/tools.ts`: additive `list_source_failures` handler and MCP registration.
- `src/doctor/service.ts`: unresolved source-failure diagnostic with actionable command.
- `tests/project-summary-code-context.test.ts`: deterministic selection, containment, exclusion, truncation, and limits.
- `tests/project-summary.test.ts`: notes, fingerprints, output hash, manual-edit refusal, and forced backup.
- `tests/storage-migrations.test.ts`, `tests/storage-source-failures.test.ts`: v8 upgrade and store lifecycle.
- `tests/conversation.test.ts`, `tests/sync.test.ts`: parsing failure persistence, deduplication, and resolution.
- `tests/cli.test.ts`, `tests/mcp-tools.test.ts`, `tests/doctor.test.ts`: public diagnostics.
- `README.md`, `MEMORY_SYSTEM.md`, `docs/index.md`, `docs/quickstart.md`, `docs/mcp-setup.md`, `docs/architecture.md`, `docs/public/architecture.html`: v0.5 behavior and operations.
